# ProjectSberF4

Веб-прототип для автоматической проверки выезда болида за пределы гоночной трассы по фото и видео с судейской камеры.

Проект объединяет несколько CV-моделей: сегментацию зоны трассы, поиск колёс, поиск автомобиля и простую геометрическую проверку нарушения. Интерфейс позволяет загрузить изображение или видео, посмотреть результат на кадре, а для видео — пролистывать обработанные кадры через встроенный просмотрщик.

## Что умеет приложение

- загружать изображение или видео через веб-интерфейс;
- обрабатывать видео на backend через OpenCV, без зависимости от события `loadedmetadata` в браузере;
- выбирать частоту выборки кадров из видео;
- показывать обработанные кадры через слайдер, кнопки «назад/вперёд» и режим воспроизведения;
- находить маску трассы;
- находить колёса болида;
- дополнительно находить сам автомобиль через отдельную YOLO-модель;
- определять участки, где колёса находятся вне разрешённой области трассы;
- использовать фиксированную маску трассы для статичной камеры, чтобы разметка трассы не прыгала от кадра к кадру.

## Идея работы

Для одного кадра приложение выполняет такую цепочку:

1. Backend получает кадр или извлекает его из видео.
2. Модель трассы строит область допустимого движения.
3. Модель колёс находит колёса болида.
4. Модель машины находит автомобиль целиком.
5. Модуль проверки сравнивает положение колёс с маской трассы.
6. Frontend рисует поверх исходного кадра маски, рамки и зоны нарушения.

Для статичных судейских камер есть отдельный режим: приложение может выбрать один наиболее удачный опорный кадр из начала видео и использовать найденную на нём маску трассы для всех следующих кадров. Это полезно, когда машины перекрывают часть трассы и мешают сегментации.

## Используемые модели

В проекте предусмотрены три модели:

| Назначение | Переменная в `.env` | Пример пути |
|---|---|---|
| Сегментация трассы | `TRACK_MODEL_PATH` | `./app/models/best_track.pt` |
| Сегментация колёс | `WHEEL_MODEL_PATH` | `./app/models/yolo26_seg_best.pt` |
| Детекция/сегментация машины | `CAR_MODEL_PATH` | `./app/models/best_cars.pt` |

Модель машины может быть не только segmentation, но и detection. Если модель возвращает только bounding boxes, backend преобразует рамку автомобиля в прямоугольный полигон, чтобы frontend всё равно мог её отрисовать.

## Быстрый запуск

Требования:

- Python 3.11 или новее;
- установленный `pip`;
- веса моделей в папке `app/models/`.

Создание окружения и установка зависимостей:

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
```

На Windows активация окружения обычно выглядит так:

```powershell
python -m venv .venv
.venv\Scripts\activate
python -m pip install --upgrade pip
pip install -r requirements.txt
```

Создайте локальный файл настроек:

```bash
cp .env-example .env
```

Пример содержимого `.env`:

```env
WHEEL_MODEL_PATH=./app/models/yolo26_seg_best.pt
TRACK_MODEL_PATH=./app/models/best_track.pt
CAR_MODEL_PATH=./app/models/best_cars.pt

CAR_MODEL_CONF=0.5
CAR_MODEL_IOU=0.7
CAR_CLASS_NAMES=car,cars,vehicle,formula,formula4

YOLO_CONFIG_DIR=/tmp/Ultralytics
```

Запуск сервера:

```bash
uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

После запуска откройте в браузере:

```text
http://127.0.0.1:8000
```

Документация FastAPI доступна здесь:

```text
http://127.0.0.1:8000/api/docs
```

## Работа с видео

Видео отправляется на backend целиком. Сервер сохраняет его во временный файл, открывает через OpenCV, выбирает кадры с заданной частотой и возвращает список обработанных кадров.

В интерфейсе можно настроить:

- FPS выборки — сколько кадров в секунду брать из видео;
- максимальное число кадров для обработки;
- режим фиксированной маски трассы;
- количество первых кадров, среди которых выбирается опорная маска трассы.

Если видео не открывается, чаще всего проблема в контейнере или кодеке. В таком случае лучше перекодировать файл в MP4/H.264:

```bash
ffmpeg -i input.mkv -c:v libx264 -pix_fmt yuv420p -movflags +faststart output.mp4
```

## Основные API endpoints

### Проверка одного кадра

```text
POST /api/infer/violation
```

Form-data параметры:

| Параметр | Описание |
|---|---|
| `frame` | файл изображения |
| `frame_index` | номер кадра |
| `offtrack_threshold` | порог мягкого нарушения |
| `hard_violation_threshold` | порог серьёзного нарушения |

### Обработка видео

```text
POST /api/infer/video
```

Form-data параметры:

| Параметр | Описание |
|---|---|
| `video` | видеофайл |
| `sample_fps` | частота выборки кадров |
| `max_frames` | ограничение на число обработанных кадров |
| `offtrack_threshold` | порог мягкого нарушения |
| `hard_violation_threshold` | порог серьёзного нарушения |
| `static_track_mode` | использовать фиксированную маску трассы |
| `track_reference_frames` | сколько первых кадров проверять для выбора маски трассы |

### Получение сырых масок кадра

```text
POST /api/infer/frame
```

Этот endpoint возвращает найденные маски без полной логики проверки нарушения.

## Настройка модели машины

Если машины не отображаются в интерфейсе, проверьте несколько моментов:

1. Файл весов действительно лежит по пути из `CAR_MODEL_PATH`.
2. В `.env` указан правильный путь, например:

   ```env
   CAR_MODEL_PATH=./app/models/best_cars.pt
   ```

3. Если модель обучалась с другим названием класса, добавьте его в `CAR_CLASS_NAMES`:

   ```env
   CAR_CLASS_NAMES=car,cars,vehicle,formula,formula4,bolid
   ```

4. Если модель слишком редко находит машины, можно временно снизить confidence:

   ```env
   CAR_MODEL_CONF=0.15
   ```

После изменения `.env` сервер нужно перезапустить.

## Что важно учитывать

- Режим фиксированной маски трассы рассчитан на статичную камеру. Для движущейся камеры его лучше отключить.
- Чем выше `sample_fps` и `max_frames`, тем дольше будет обрабатываться видео.
- Большие `.pt` файлы лучше хранить через Git LFS, чтобы GitHub не отклонял push из-за размера файла.
- `.env` и виртуальные окружения не нужно загружать в репозиторий. Для этого они должны быть прописаны в `.gitignore`.
